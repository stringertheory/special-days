# Security policy

If you've found a security issue in `special-days`, please report it
privately using GitHub's
[Report a vulnerability](https://github.com/stringertheory/special-days/security/advisories/new)
button on the Security tab. That goes straight to me and isn't visible
publicly until I publish an advisory.

I'll acknowledge within a week and either ship a fix, document the
limitation, or explain why the report falls outside the package's
threat model.

Please don't disclose security issues in public GitHub issues or PRs —
both are world-readable from the moment they're opened.
